System Overview

Figure 1. Our visual interface: (top-left) control zone, (top) TimeArc, (top-right) St. Himark map and (bottom) message table

Control zone includes
  • Search box: for searching user, location or category
  • Category legend: click on category name to show or hide that category
  • Mentioned together slider: adjust lower limit of links
TimeArc have 2 main components: the stream graph of terms/topics and the link between 2 topics present the connection. The thicker of link display the stronger connection between them. This connection has been calculate by how coincident 2 world present in the same message.
The data using in this application is processed directly on web. The process sequence includes removing duplicated post (such as repeated posts) and spam (sale promotion), extract keyworks basing on category definition and aggregate post content by hour.

The data using in this application is an array of message and have been process directly from web browser. In the real situation with larger dataset, the best solution is using web server which process the data before sending the extract terms to client via web api. Moreover, the accuracy of this method can be improve if using server rather than using client browser. However, whether using server or not, our web application still have ability to analysis static and dynamic data.
